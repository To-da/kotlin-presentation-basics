Why Kotlin ?
Motivation

Created by JetBrains

For their internal purposes.
... scala compilation was too slow ...

They work with various languages on daily basis.
... should be able to design one ...

Great IDE support and tooling.
... IntellyJ IDEA ...

Mature

Version 1.3 now.

No more breaking changes in API (since 1.0).

Defined process for components stability.

Adopters

Server-side - Spring, Ratpack* ...

Multiplatform

Kotlin Philosophy

Pragmatic

Designed to solve real-world problems.
... based on industry experience - Tooling creators + years of usage early versions by adopters ...

Do not enforce any particular programming style or paradigm.
... can be used just like Java (OOP/imperative) in the beginning ...

Language features designed with tooling in mind.
... IDE find more concise constructs, fix the problems etc ...

Concise

“make you code smaller”
... less code means fewer bugs :) ...
... developer spent (much) more time reading existing code then writing new one ...

Code contains meaning.
... less boilerplate (getters/setters/constructors etc.) ...

No “free” operator overloading.
... you can always find what you want in documentation ...
... no cryptic punctuation sequences ( hello Scala :) ) ...

Safe

Preventing certain kinds of errors.
... more than Java ...

no NullPointerException / better ways to handle nullable data.

You can debug code between Kotlin and Java (mixed code-bases)
